Fuego API Gateway

All endpoints served via xfg-api REST gateway at http://localhost:8787/v1. Point the gateway at any Fuego daemon with CORE_RPC_URL=http://your-node:18180 node server.js. Fuego v1.10.00 "AzorAhai"+ required for HⲶ∆T/AMM endpoints.

Node Info

  • GET /v1/node/info
    Network height, difficulty, hashrate, deposit count, fee address, version
  • GET /v1/node/height
    Current block height
  • GET /v1/node/blockcount
    Total block count (JSON-RPC)
  • GET /v1/node/last_block_header
    Most recent block header (reward, difficulty, hash, timestamp)
  • GET /v1/node/block_header_by_height/:height
    Block header for a specific height

Blocks & Transactions

  • GET /v1/node/block/:hash
    Full block data by hash (transactions, amounts, sizes)
  • GET /v1/node/transaction/:hash
    Full transaction details by hash
  • GET /v1/node/blocks/list
    Recent blocks list (mempool-friendly summary)
  • GET /v1/node/pool/transactions
    Unconfirmed transactions in mempool
  • POST /v1/node/reserve-proof
    Verify reserve proof signature for balance attestation

HⲶ∆T Flatcoin

  • GET /v1/node/heat/metrics
    HⲶ∆T supply, burned XFG, redemption price, redemption rate, PI controller state, treasury balance
  • GET /v1/node/eternal-flame
    Total XFG permanently burned via Burn2Mint (EternalFlame reserve)

Hearth AMM (HⲶ∆T/XFG)

  • GET /v1/node/amm/pool
    Pool reserves (XFG/HⲶ∆T), constant-product K, spot price, LP shares, accumulated fees
  • GET /v1/node/amm/quote?from=XFG&to=HEAT&amount=1000
    Simulate swap output on the Hearth AMM (1% fee applied)

Fee Pool & Treasury

  • GET /v1/node/fee-pool
    Swap fee pool balance, distribution split (80% CDs / 20% Treasury), epoch info
  • GET /v1/node/epoch/history
    Historical epoch data — fee generation, CD yield distributed, treasury allocations
  • GET /v1/node/treasury
    Treasury wallet balance and activity

Commitment Deposits (CDs)

  • GET /v1/node/deposits
    All active deposits on the network
  • GET /v1/node/cd/yield?amount=1000&term=900
    Estimate CD yield for a given XFG amount and term (in blocks)
  • GET /v1/node/maturing-deposits
    Deposits nearing maturity (eligible for rollover)

Aliases

  • GET /v1/node/alias/:name
    Lookup alias by name (returns address, type, block, status)
  • GET /v1/node/alias/by-address/:address
    Find alias registered to an address
  • GET /v1/node/aliases
    List all registered aliases

Swaps & Pricing

  • GET /v1/node/swaps/price
    TWAP, seed rate, composite rate, per-source oracle breakdown, Hearth ratio, HⲶ∆T USD, XFG USD range
  • GET /v1/node/swaps/offers?from=BTC&to=XFG
    Active atomic swap offers by pair
  • GET /v1/node/swaps/trades?limit=50
    Recent atomic swap trades

ZK Bridge / Commitments

  • GET /v1/node/commitment/stats
    HⲶ∆T vs COLD commitment counts, total locked
  • GET /v1/node/commitment/merkle-root
    Current commitment index merkle root
  • GET /v1/node/commitment/proof/:id
    Merkle proof for a specific commitment (bridge validation)

Direct Daemon (no gateway needed)

  • GET /getinfo
    Native daemon info — works directly against fuegod:18180
  • GET /getheight
    Native daemon height
  • GET /peers
    Connected peer list
  • GET /getethereal
    Total burned XFG (EternalFlame)
  • GET /paymentid
    Generate a random payment ID
  • POST /json_rpc
    JSON-RPC 2.0 gateway — all core methods (f_block_json, f_transaction_json, check_reserve_proof, getblockcount, getlastblockheader, etc.)
